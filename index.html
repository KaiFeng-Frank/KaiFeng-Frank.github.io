Kai Feng

ME Postgraduate, The University of Hong Kong

I build reproducible ROS2 SLAM and perception infrastructure for RGB-D and LiDAR-Inertial-Camera robots, with current work on ORB-SLAM3 wheel fusion and Gaussian-SLAM reproduction.

GitHub LinkedIn

My work turns research SLAM code into ROS2 components that mapping and planning modules can swap, validate, and debug. I care about interface contracts, trajectory evidence, and failure signals, not only whether a demo runs once.

I am interested in research collaboration around ground/mobile robot localization, active mapping, continuous-time odometry, and reproducible robotics systems.

Research Focus

  • Swappable localization backends for active mapping systems
  • ORB-SLAM3 RGB-D localization with wheel-odometry fusion
  • Continuous-time LiDAR-Inertial-Camera odometry on ROS2 Jazzy
  • Gaussian-SLAM reproduction, validation, and render-feedback studies

Selected Projects

ORB-SLAM3 Wheel ROS2

ROS2 Jazzy, ORB-SLAM3, GTSAM, wheel odometry

Problem
Make ORB-SLAM3 usable as a replaceable ROS2 localization backend for active mapping and planning.
Contribution
Links against unmodified libORB_SLAM3.so, adds GTSAM fixed-lag wheel/visual smoothing, and publishes odometry, keyframes, camera intrinsics, and reliability topics.
Evidence
Official ORB-SLAM3 front-end reproduces TUM fr1_xyz ATE 0.0102 m; the ROS2 node measures 0.0129 m end-to-end.

Repository

Coco-LIC ROS2 Jazzy

ROS2 Jazzy, continuous-time odometry, rosbag2

Problem
Port a ROS1/catkin continuous-time LIC odometry system to ROS2 Jazzy without losing timing and bag replay semantics.
Contribution
Provides a ROS2 workspace with Ceres 2.2 compatibility, rosbag2 input, per-point LiDAR timing support, and validation packaging.
Evidence
FAST-LIVO2 CBD_Building_01 validation reports 2.12 cm LIO and 2.27 cm LICO ATE RMSE over 1231 / 1231 matched poses.
Code
Repository · Validation

Gaussian-LIC ROS2

ROS2 Jazzy, LiDAR-Inertial-Camera SLAM, Gaussian mapping infrastructure

Problem
Turn a large LIC + Gaussian mapping research stack into a ROS2 checkpoint with explicit reproduction boundaries.
Contribution
Packages middleware contracts, continuous-time tracking surfaces, CUDA/Torch Gaussian mapping, dataset profiles, and audit scripts in one ROS2 Jazzy workspace.
Evidence
Public reports cover FAST-LIVO, FAST-LIVO2, M2DGR, MCD, and R3LIVE; GL2 results include CBD full-sequence tracking and PSNR/ATE ablations.
Code
Repository · GL2 results · Matrix

Autonomous Logistics Robot

Raspberry Pi 5, ESP32-S3, ROS2, Nav2, QR vision

Problem
Build an indoor delivery robot that ties navigation, perception, motor control, and payload handling into one ROS2 system.
Contribution
Integrates Raspberry Pi 5, ESP32-S3 micro-ROS motor control, Nav2, QR perception, mecanum drive, and a lift mechanism.
